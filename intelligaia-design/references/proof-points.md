# Proof points — the canonical case source

The **single source of truth** for every Intelligaia client case, metric, and
proof point. Voice rules, proposals, decks, the website Work page, and outreach
all cite from here. Never quote a metric from memory; never invent or stretch one.

## How to use across media

| Medium / artifact | How many cases | How to render |
|---|---|---|
| Cold outreach message | **One** — the single most analogous | Inline, one metric |
| Proposal / RFP response | **Up to 2** — most analogous first | `proof_editorial` block: narrative + metric wall |
| Capability / pitch deck | 3–6 across the deck | One case per slide; lead with the metric |
| Case study (standalone) | 1 | Full entry, outcome metric as hero |
| Website Work page | All cleared cases | One card per case (`metric` + descriptor) |
| One-pager | 3–4 proof points | Metric row, Trirong italic |

### Selection rules
1. **Match vertical first.** A relevant mid-market case beats an impressive but
   unrelated enterprise one.
2. **Match the metric to the audience.** ROI / scale for executives; speed,
   architecture, execution for builders.
3. **Never stretch.** No analogous case → use the closest adjacent and say so.
4. **Cisco cases are the strongest anchors** for AI, enterprise IT, and sales-ops
   contexts — reserve them for the highest-fit situations; don't overuse.
5. **Name clearance.** Cases marked *not cleared* must be described by sector, not
   name, in public-facing output.
