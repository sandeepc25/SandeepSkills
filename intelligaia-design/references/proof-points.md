# Proof points — the canonical case source

The single source of truth for every Intelligaia client case, metric, and proof point. Voice rules,
proposals, decks, the website Work page, and outreach all cite from here. Never quote a metric from
memory; never invent or stretch one. (Unchanged from v1 — reproduced here so the v2 package is
self-contained.)

## How to use across media

| Medium / artifact | How many cases | How to render |
|---|---|---|
| Cold outreach message | One — the single most analogous | Inline, one metric |
| Proposal / RFP response | Up to 2 — most analogous first | Narrative + metric wall |
| Capability / pitch deck | 3-6 across the deck | One case per slide; lead with the metric |
| Case study (standalone) | 1 | Full entry, outcome metric as hero |
| Website Work page | All cleared cases | One card per case |
| One-pager | 3-4 proof points | Metric row, Trirong italic |

### Selection rules
1. Match vertical first. 2. Match the metric to the audience. 3. Never stretch — use the closest
adjacent case and say so. 4. Cisco cases are the strongest anchors for AI/enterprise IT/sales-ops —
reserve for the highest-fit situations. 5. Name clearance: describe by sector if not cleared.
