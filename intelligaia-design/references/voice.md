# Voice — editorial reference (v2 — unchanged from v1)

Applies to all Intelligaia copy in every medium. Proof points and metrics live in
`references/proof-points.md`. This file governs *how* Intelligaia sounds.

## Default register
**Confident · Technical · Outcome-focused · Direct.** Sounds like a firm that has built real things
at Cisco scale and doesn't need to prove it — not a consultancy with a methodology deck.

> Test: would a VP of Product at an enterprise read this and think "these people have done this
> before"? If not, rewrite.

## Vocabulary — always / never

| Use this | Not this |
|---|---|
| Workshops | Discovery / Kickoff / Strategy sessions |
| Labs | POC phase / Prototype sprint / R&D |
| Studio | Build phase / Delivery / Execution |
| AI-centred | AI-powered / AI-enabled / AI-driven |
| Design and engineering | Design then engineering / UX then dev |
| Proof-of-concept | MVP (unless the client uses it) / Pilot |
| Production-ready | Launched / Shipped / Go-live |
| Enterprise product | Solution / Platform (only if the client calls it that) |
| Design system | UI kit / Component library (unless technically accurate) |
| Outcome | Result / Deliverable |
| Multi-agent / RAG | AI chatbot (unless technically accurate) |
| Engagement | Project / Contract / SOW |
| Conversation | Discovery call / Sales call |

**v2 note:** Workshops/Labs/Studio remain the correct vocabulary *when describing a staged
engagement that's actually true* — don't force every document (a one-pager, a single case study) to
walk through all three stages if it doesn't need to.

## Sentence-level rules
- Outcomes first, process second.
- Specific over impressive: "3x deal-cycle speed" beats "dramatically faster".
- No stated assumptions, no flattery.
- No AI-tell phrases: "in today's rapidly evolving AI landscape", "leverage cutting-edge", "robust
  solution", "holistic approach", "seamless", "game-changing", "unlock", "ever-evolving".
- Short active sentences. One idea per sentence.

## Authoritative service copy
**Workshops** — Structured sessions moving teams from ambiguity to direction, fast. Discovery
workshops, AI opportunity mapping, design sprints, roadmap definition. Output is a concrete plan.

**Labs** — Where ideas get tested before they get funded. POCs, AI prototypes, technical validation
at speed. Most Labs engagements run 8–12 weeks.

**Studio** — End-to-end product design and engineering, running simultaneously — not handed off.

### Core services
UX/UI Design · AI POC Development · Front-End Engineering · Design Systems · AI Implementation ·
Data Visualisation · Discovery Workshops.

## Positioning copy

**Positioning statement:** Intelligaia is an AI-centred design and engineering firm. We take AI
proofs-of-concept to production-ready solutions — owning the outcome from the first Workshop to
final deployment.

**What makes us different:** Most firms deliver a technical build. We own the outcome: design and
engineering work together so that what ships gets used — by real users, at enterprise scale.

**Process, in one line:** Workshops establish direction. Labs validate what's possible. Studio
builds it. Each stage commits to a concrete outcome.
